# Eleanor PHP Library

Современные PHP-фреймворки постепенно превратились в огромные экосистемы, требующие установки, настройки сервера, роутинга, build-пайплайнов, dependency-графов и целой инфраструктуры для запуска приложения. Для многих проектов подобный подход является избыточным.

Eleanor PHP Library придерживается другой философии. Библиотека предназначена для разработчиков, которым нужен лёгкий, практичный, прозрачный и независимый от фреймворков PHP-код без скрытой магии и чрезмерных слоёв абстракций.

Без обязательной установки.  
Без навязанной router-архитектуры.  
Без framework lifecycle magic.  
Без сложной конфигурации сервера.

Просто подключи библиотеку и пиши PHP.

---

## Принципы проектирования

### 1. Установка не требуется

Библиотека работает сразу после подключения. Файлы можно просто скопировать в проект и начать использовать.

Установка через Composer является необязательной:

```bash
composer require sunvas/eleanor-php-library
```

---

### 2. Совместимость с обычным хостингом

Библиотека специально проектируется так, чтобы работать в традиционных условиях виртуального хостинга без необходимости:

- настройки rewrite engine;
- CLI bootstrap-скриптов;
- daemon-процессов;
- контейнеризации;
- привилегированного доступа к серверу.

---

### 3. Лёгкая архитектура

Цель библиотеки — не заменить PHP очередным языком абстракций. Eleanor PHP Library помогает разрабатывать приложения, сохраняя код читаемым и предсказуемым.

Без скрытых dependency injection контейнеров.  
Без непрозрачного lifecycle management.  
Без многослойных framework internals.

---

### 4. Базовые возможности уже встроены

Библиотека уже включает:

- шаблонизатор;
- cache-машины;
- MySQL-драйвер;
- подсистему локализации;
- отложенную инициализацию объектов;
- структурированное логирование ошибок;
- гармоничную интеграцию сторонних классов, интерфейсов, трейтов и перечислений.

---

### 5. Отложенное создание объектов

Объекты могут создаваться лениво. Например, соединение с MySQL не устанавливается до момента фактической необходимости.

---

### 6. Структурированное логирование ошибок

Подсистема логирования автоматически поддерживает порядок в логах:

- одинаковые ошибки группируются;
- старые логи автоматически архивируются.

---

## Философия

Eleanor PHP Library не пытается диктовать структуру приложений. Это набор инструментов, а не framework.

Главная задача библиотеки проста:

- сохранить PHP лёгким;
- сохранить архитектуру понятной;
- сохранить исходный код читаемым.

Без головоломок под капотом.

---

## Требования

- PHP 8.5 или выше
