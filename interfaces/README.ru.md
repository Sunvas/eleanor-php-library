# Каталог PHP-интерфейсов

Данный каталог предназначен для размещения PHP-интерфейсов, используемых в библиотеке Eleanor PHP Library.

## Принцип работы автозагрузчика

Автозагрузчик обслуживает только пространство имён `Eleanor\Interfaces`. При запросе интерфейса автозагрузчик преобразует имя интерфейса в имя файла, приводя его к нижнему регистру и дополнительно проверяя kebab-case вариант имени. Например, запрос:

```php
Eleanor\Interfaces\TestInterface
```

приведёт к последовательной проверке файлов:

```text
testinterface.php
test-interface.php
```

Поддерживается организация интерфейсов через подкаталоги. Каждый подкаталог соответствует дополнительному уровню пространства имён внутри `Eleanor\Interfaces`. Например, для интерфейса:

```php
Eleanor\Interfaces\Test\SomeFeatureInterface
```

автозагрузчик выполнит поиск файлов:

```text
./test/somefeatureinterface.php
./test/some-feature-interface.php
```

---

## Правила и требования

### 1. Именование файлов

Автозагрузчик поддерживает оба варианта имён файлов:

```text
testinterface.php
test-interface.php
```

Имена файлов регистрозависимы и должны быть записаны в нижнем регистре.

### 2. Независимость от namespace и имени интерфейса

Автозагрузчик определяет соответствие интерфейса исключительно по имени файла. Фактическое пространство имён и имя интерфейса внутри файла не участвуют в поиске и могут быть произвольными. Благодаря этому каталог может использоваться как адаптерный слой для сторонних библиотек или внешних интерфейсов.

### 3. Возврат полного имени интерфейса (FQCN)

Если интерфейс внутри файла не соответствует ожидаемому имени интерфейса в `Eleanor\Interfaces`, файл должен вернуть полное квалифицированное имя интерфейса (FQCN):

```php
return \Vendor\Package\SomeInterface::class;
```

Пример:

```php
<?php
namespace Vendor\Package;

interface SomeInterface
{
    public function execute():void;
}

return \Vendor\Package\SomeInterface::class;
```

После загрузки файла автозагрузчик:

- использует интерфейс напрямую, если он уже принадлежит пространству имён `Eleanor\Interfaces`;
- либо автоматически создаёт псевдоним внутри `Eleanor\Interfaces`.

### 4. Автоматическое создание псевдонима

Интерфейс не обязан принадлежать пространству имён `Eleanor\Interfaces`. Если файл возвращает FQCN интерфейса из другого пространства имён, автозагрузчик автоматически создаёт псевдоним внутри `Eleanor\Interfaces`, обеспечивая доступ к интерфейсу через единый механизм автозагрузки. Например:

```php
Eleanor\Interfaces\Test\SomeFeatureInterface
```

может фактически ссылаться на:

```php
\External\Library\FeatureInterface
```

### 5. Необязательный `return`

Если интерфейс уже объявлен в ожидаемом пространстве имён и его имя соответствует имени файла, файл может не возвращать FQCN. Например, файл:

```text
test/some-feature-interface.php
```

может напрямую содержать:

```php
<?php
namespace Eleanor\Interfaces\Test;

interface SomeFeatureInterface
{
    public function execute():void;
}
```

В этом случае автозагрузчик обработает интерфейс автоматически без дополнительного `return`.
