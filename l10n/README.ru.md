# Файлы локализации

Данный каталог содержит языковые (l10n) файлы компонентов библиотеки. Каждый файл локализации должен возвращать массив, где значениями могут быть:

- строки — для простых случаев перевода;
- Closure — для динамических или структурно сложных переводов.

Пример:

```php
return [
	'title'=>'Профиль пользователя',

	'items'=>fn(int $count)=>match($count)
	{
		0=>'Нет элементов',
		1=>'Один элемент',
		default=>"$count элементов",
	},
];
```

Имя каждого файла должно содержать суффикс с кодом языка. Поддерживаются следующие форматы имён файлов:

```text
[название-компонента].[код-языка].php
[код-языка].php
```

Примеры:

```text
auth.en.php
auth.ru.php
en.php
ru.php
```

## Почему переводы являются программируемыми

Система локализации намеренно не ограничивается gettext-подходом, основанным исключительно на функциях вроде `__()` и статических строках перевода. Подобные решения могут быть удобны для простых подстановок, однако быстро становятся недостаточными при работе со сложными языковыми конструкциями, повторным использованием переводов, динамическим формированием сообщений и языковыми правилами. Поэтому в Eleanor PHP Library переводы представлены не статическими словарями, а исполняемыми PHP-описаниями. Это позволяет реализовать:

- повторное использование переводов;
- наследование и композицию;
- динамическую генерацию сообщений;
- языковые грамматические правила;
- сложные формы множественного числа;
- контекстно-зависимое форматирование.

## Пример сложного случая для gettext

Предположим, имеются две переменные: количество яблок и количество апельсинов. Полноценная система локализации может потребовать формирование следующих сообщений:

- `Ничего нет`;
- `Есть только 1 яблоко`;
- `Есть только 5 яблок`;
- `Есть 7 яблок и 1 апельсин`;
- `Есть 10 апельсинов и 3 яблока`;
- `Есть по 5 яблок и апельсинов`.

Подобные конструкции крайне неудобно выражать через статические gettext-правила, особенно когда в одном сообщении одновременно участвуют несколько независимых форм множественного числа. Для языков со сложной грамматикой, таких как русский, задача становится ещё сложнее.
